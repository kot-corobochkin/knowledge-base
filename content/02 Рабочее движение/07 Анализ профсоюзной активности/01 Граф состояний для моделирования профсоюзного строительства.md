# Идея: граф состояний для моделирования профсоюзного строительства

Можно представить процесс создания профсоюза как **граф состояний (state graph)** или **граф действий**.  
В этом графе:

- **узлы (nodes)** — это состояния организации работников;
- **ребра (edges)** — это возможные действия;
- **веса (weights)** — влияние действия на успех или риск;
- **цель (goal node)** — успешное создание профсоюза.

Фактически это похоже на систему поиска пути, как в AI или экспертных системах.

---

# 1. Основные элементы системы

## 1. Начальное состояние

Например:

```
Компания: Amazon  
Работников: 3  
Профсоюз: отсутствует  
Уровень организации: 0  
Риск репрессий: низкий

```
Это стартовый узел графа.

---

## 2. Целевое состояние

Конечный узел:

Создан профсоюз  
Есть лидерская группа  
Есть поддержка большинства  
Есть юридическая регистрация

Это **goal node**.

---

# 3. Узлы графа

Каждый узел описывает ситуацию:

Пример:

```
Состояние 1  
3 работника обсуждают проблему  
  
Состояние 2  
есть ядро активистов  
  
Состояние 3  
есть поддержка 10+ сотрудников  
  
Состояние 4  
подача на регистрацию профсоюза
```

---

# 4. Действия (ребра графа)

Это ключевая часть твоей идеи.

Пример действий:

### Безопасные действия

поговорить с коллегой  
создать чат  
обсудить проблемы  
собрать встречу  
написать коллективное письмо

Пример веса:

+0.2 рост организации  
+0.4 рост доверия  
+0.5 расширение сети

---

### Усиливающие действия

создать комитет  
найти лидеров  
связаться с профсоюзной организацией

Вес:

+1.0  
+1.3  
+1.7

---

### Рискованные действия

Например:

публичный протест  
конфликт с менеджментом  
утечка информации

Вес:

+0.5 к росту  
НО  
+1.2 к риску

---

### Запрещенные или опасные действия

Ты правильно предложил:

действие → штраф

Например:

минус 7  
возможный арест лидеров  
разрушение сети  
увольнения

Это ведет к другому графу:

граф репрессий

---

# 5. Условия действий

Важная часть системы — **предусловия**.

```
Пример:

собрание работников  
возможно только если  
есть минимум 3 активиста

или

публичная кампания  
возможна если  
есть поддержка 30% работников
```

---

# 6. Динамика графа

Твой граф — это не просто граф, а:

**динамическая модель развития организации**

каждое действие изменяет:

- уровень организации
- уровень доверия
- риск
- скорость роста

---

например:

Amazon union campaigns

И на основе этих данных: обучить веса действий.

# 8. Как система будет искать путь

Можно использовать алгоритмы:

### Поиск кратчайшего пути

A*

максимум успеха

# 9. Фактически это гибрид

Твоя идея — это комбинация:

1 графа состояний  
2 экспертной системы  
3 симуляции социальных процессов  
4 модели стратегии

очень интересная архитектура.

# 10. Как это может выглядеть формально

Состояние:

```
S = {  
workers: 3  
support: 0.1  
leaders: 0  
risk: 0.2  
visibility: low  
}
```

Действие:

A = talk_to_colleague

Переход:

S → S'

Изменение:

support +0.1  
risk +0.02  
network +1
