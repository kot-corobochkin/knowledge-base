## 7. Tarantool and Picadata
- **Tarantool** was designed for use cases where code needs to run close to data (low‑latency, in‑memory computing).
- It pioneered the **per‑core sharding** architecture (one process per core), which is now common in modern runtimes (Tokio, Go, etc.).
- **Picadata** (Konstantin’s current company) is built on this same sharded, per‑core architecture but:

Written in **Rust** (performance, safety).
- Uses **PostgreSQL wire protocol** and **PostgreSQL syntax**.
- Supports **plugins** (extensions) written in Rust.
- Can run as a **sharded, in‑memory PostgreSQL** for OLTP workloads with high concurrency.
