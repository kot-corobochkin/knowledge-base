# 📚 Теория: команда `journalctl -u node_js.service -n 100 --no-pager`

## 🎯 Что это за команда

**`journalctl`** - это утилита для просмотра логов **systemd** (системного менеджера Linux). Она позволяет читать логи всех сервисов, ядра и системы в целом.

### Визуальное представление:
```
┌─────────────────────────────────────────────────────────────────┐
│  journalctl - это "системный черный ящик" Linux                 │
│                                                                  │
│  Все, что пишут сервисы → собирается в journal → читается через │
│  journalctl                                                     │
└─────────────────────────────────────────────────────────────────┘
```

## 📊 Разбор команды по частям

```bash
journalctl -u node_js.service -n 100 --no-pager
```

| Часть команды | Значение | Что делает |
|---------------|----------|-------------|
| `journalctl` | Базовая команда | Запуск просмотрщика логов |
| `-u node_js.service` | `--unit=` | Фильтр по конкретному сервису |
| `-n 100` | `--lines=` | Показать последние 100 строк |
| `--no-pager` | Без постраничного вывода | Показать весь вывод сразу (без `less`) |

## 🔬 Детальный разбор

### 1. **`journalctl` - базовая команда**

**Что такое journal?**
- Централизованное хранилище логов в Linux
- Собирает логи от: systemd, сервисов, ядра, приложений
- Хранится в бинарном формате (не текстовые файлы)

```bash
# Без аргументов - показывает ВСЕ логи системы
journalctl
# Результат: тысячи строк со всех сервисов
```

### 2. **`-u node_js.service` - фильтр по сервису**

```bash
# Без -u: все логи системы
journalctl | wc -l
# 50000 строк

# С -u: только логи конкретного сервиса
journalctl -u node_js.service | wc -l
# 100 строк (только ваш сервис)
```

**Альтернативные формы:**
```bash
journalctl -u node_js.service     # короткая форма
journalctl --unit=node_js.service # длинная форма
```

### 3. **`-n 100` - ограничение количества строк**

```bash
# Все строки (может быть миллион)
journalctl -u node_js.service

# Только последние 100 строк (быстро)
journalctl -u node_js.service -n 100

# Последние 50 строк
journalctl -u node_js.service -n 50

# Последние 200 строк
journalctl -u node_js.service -n 200
```

### 4. **`--no-pager` - отключение постраничного вывода**

**С pager (по умолчанию):**
```bash
journalctl -u node_js.service
# Открывается в less (навигация стрелками, q - выход)
```

**Без pager:**
```bash
journalctl -u node_js.service --no-pager
# Весь вывод сразу в терминал (как cat)
```

## 📊 Сравнение с традиционными логами

| Аспект | Традиционные логи | journalctl |
|--------|------------------|------------|
| **Формат** | Текстовые файлы (`/var/log/`) | Бинарная БД |
| **Поиск** | `grep`, `awk` | Встроенные фильтры |
| **Фильтрация** | Сложная | Простая (`-u`, `-p`, `--since`) |
| **Ротация** | `logrotate` | Автоматическая |
| **Просмотр** | `cat`, `tail`, `less` | `journalctl` |
| **Структура** | Неструктурированный | Структурированный (JSON) |

### Пример сравнения:

```bash
# Традиционный способ (syslog)
tail -n 100 /var/log/syslog | grep node_js

# journalctl способ
journalctl -u node_js.service -n 100
```

## 🛠️ Дополнительные возможности journalctl

### Фильтрация по времени:
```bash
# За последние 10 минут
journalctl -u node_js.service --since "10 minutes ago"

# За последний час
journalctl -u node_js.service --since "1 hour ago"

# Конкретный диапазон
journalctl -u node_js.service --since "2024-04-10 10:00:00" --until "2024-04-10 12:00:00"

# Сегодня с утра
journalctl -u node_js.service --since today
```

### Фильтрация по уровню ошибок:
```bash
# Только ошибки
journalctl -u node_js.service -p err

# Уровни: emerg, alert, crit, err, warning, notice, info, debug
journalctl -u node_js.service -p warning
```

### Вывод в JSON (для парсинга):
```bash
# JSON формат
journalctl -u node_js.service -o json

# Только сообщения
journalctl -u node_js.service -o json | jq '.MESSAGE'
```

### Следование за логами (как tail -f):
```bash
# Следить в реальном времени
journalctl -u node_js.service -f

# Как tail -f, но с фильтрацией
journalctl -u node_js.service -f -n 50
```

## 📚 Проверочные вопросы

### Вопрос 1
Что произойдет при выполнении команды `journalctl -u nginx.service -n 20 --no-pager`?

**A)** Откроется редактор vim с логами nginx  
**B)** Покажет последние 20 строк логов nginx и сразу выйдет  
**C)** Покажет все логи nginx в постраничном режиме  
**D)** Очистит логи nginx

<details><summary><b>Ответ:</b></summary>

**B** - `-n 20` ограничивает вывод 20 строками, `--no-pager` отключает постраничный вывод
</details>

---

### Вопрос 2
Какой командой можно следить за логами сервиса в реальном времени (как `tail -f`)?

**A)** `journalctl -u сервис --follow`  
**B)** `journalctl -u сервис -f`  
**C)** `journalctl -u сервис --since "now"`  
**D)** `journalctl -u сервис -r`

<details><summary><b>Ответ:</b></summary>

**A и B** - Оба варианта правильные. `-f` это короткая форма `--follow`
</details>

---

### Вопрос 3
Что означает опция `-p err` в команде `journalctl -u app.service -p err`?

**A)** Показать логи за последний час (error period)  
**B)** Показать только ошибки (priority error)  
**C)** Показать логи с PID ошибки  
**D)** Показать ошибки в формате JSON

<details><summary><b>Ответ:</b></summary>

**B** - `-p` это `--priority=`, `err` означает уровень ошибки (error). Покажет только сообщения уровня ERROR и выше.
</details>
