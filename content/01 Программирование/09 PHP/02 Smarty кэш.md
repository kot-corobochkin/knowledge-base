## Теория: Очистка кеша шаблонизатора Smarty и ViewModel

Представьте, что вы печете пирожные в кондитерской. У вас есть:
- **Рецепт** — это шаблон (template)
- **Готовое пирожное** — это результат работы шаблона с данными
- **Холодильник с готовыми пирожными** — это кеш

---

### 1. Что такое Smarty?

**Smarty** — это шаблонизатор для PHP. Он отделяет логику приложения от представления (HTML/CSS).

```php
// Пример шаблона Smarty (template.tpl)
<h1>Привет, {$username}!</h1>
<ul>
{foreach $items as $item}
    <li>{$item.name}</li>
{/foreach}
</ul>
```

---

### 2. Два типа кеша в Smarty

| Тип кеша | Что это | Аналогия |
|----------|---------|----------|
| **Cache** (кеш готовых страниц) | Сохраненный готовый HTML-результат | Готовое пирожное в холодильнике |
| **Compiled Template** (скомпилированный шаблон) | PHP-код, в который Smarty превратил ваш шаблон | Рецепт, переписанный на понятном кондитеру языке |

#### Подробнее:

**Compiled Template (скомпилированный шаблон):**
- Smarty читает `.tpl` файл
- Превращает его в обычный PHP-файл
- Этот PHP-файл сохраняется в папку `templates_c`
- При следующем запросе Smarty использует уже готовый PHP-файл (быстрее, чем снова парсить .tpl)

**Cache (кеш страниц):**
- После выполнения скомпилированного шаблона с конкретными данными
- Smarty может сохранить итоговый HTML
- При следующем запросе с теми же параметрами — отдаст HTML из кеша, даже не запуская PHP-логику

---

### 3. Что делают методы очистки?

```php
$view->smarty->clearAllCache();      // Удаляет все готовые HTML-страницы из кеша
$view->smarty->clearCompiledTemplate(); // Удаляет все скомпилированные PHP-версии шаблонов
```

#### Когда это нужно?

| Ситуация | Что очищать |
|----------|-------------|
| Изменили верстку шаблона `.tpl` | `clearCompiledTemplate()` — чтобы Smarty пересобрал PHP-версию |
| Изменили данные, но старый HTML всё еще в кеше | `clearAllCache()` |
| После обновления сайта на сервере | Оба метода |
| При смене дизайна/темы | `clearCompiledTemplate()` |
| При отладке (видите старые данные) | Оба метода |

---

### 4. Что такое ViewModel?

**ViewModel** — это паттерн проектирования (часть MVVM). Простыми словами:

> Это объект, который подготавливает данные специально для отображения в представлении (шаблоне).

```php
// Без ViewModel (данные "сырые" — передаете прямо из БД)
$smarty->assign('user', [
    'first_name' => 'Иван',
    'last_name' => 'Петров',
    'birth_date' => '1990-05-15'
]);
// В шаблоне приходится форматировать: {$user.last_name} {$user.first_name}

// С ViewModel (данные уже подготовлены для вывода)
class UserViewModel {
    public $fullName;      // "Петров Иван"
    public $age;           // 34
    
    public function __construct($userData) {
        $this->fullName = $userData['last_name'] . ' ' . $userData['first_name'];
        $this->age = date('Y') - date('Y', strtotime($userData['birth_date']));
    }
}

$viewModel = new ViewModel($userData);
$smarty->assign('user', $viewModel);
// В шаблоне просто: {$user.fullName}, возраст: {$user.age}
```

---

### 5. Схема работы

```
Запрос пользователя
       ↓
[Контроллер]
       ↓
$viewModel = new ViewModel() ← подготавливает данные для отображения
       ↓
$smarty->assign() ← передает данные в шаблон
       ↓
Smarty проверяет:
   - Есть ли кеш HTML? → ДА → отдать HTML из кеша (быстро)
   - НЕТ → проверяет скомпилированный шаблон
       - Есть? → использовать
       - Нет? → скомпилировать .tpl → в PHP
       ↓
   Выполнить PHP, подставить данные → HTML
       ↓
   Сохранить HTML в кеш (если включено)
       ↓
Отдать пользователю
```

---

### 6. Полный рабочий пример

```php
class PageController {
    public function show($request) {
        // 1. Получаем данные из БД или API
        $products = $this->productRepository->getAll();
        
        // 2. Создаем ViewModel для подготовки данных
        $viewModel = new ViewModel();
        $viewModel->pageTitle = "Каталог товаров";
        $viewModel->items = $products;
        $viewModel->itemCount = count($products);
        
        // 3. Передаем в Smarty
        $view->smarty->assign('page', $viewModel);
        
        // 4. Очищаем кеш после обновления данных (если нужно)
        if ($request->isPost()) {
            $view->smarty->clearAllCache();
            $view->smarty->clearCompiledTemplate();
        }
        
        // 5. Отображаем шаблон
        return $view->smarty->display('catalog.tpl');
    }
}
```

## 3 Проверочных вопроса

### Вопрос 1 (базовый)
**В чем разница между `clearAllCache()` и `clearCompiledTemplate()`?**

<details>
<summary>Ответ</summary>

- `clearAllCache()` — удаляет готовые HTML-страницы (результат работы)
- `clearCompiledTemplate()` — удаляет скомпилированные PHP-версии шаблонов (промежуточный код)

**Аналогия:** первый удаляет готовые пирожные, второй — переписанный рецепт на понятном языке.
</details>

---

### Вопрос 2 (ситуационный)
**Вы обновили файл шаблона `header.tpl`, но на сайте всё еще отображается старая версия. Какую очистку нужно выполнить и почему?**

<details>
<summary>Ответ</summary>

Нужно выполнить `clearCompiledTemplate()`.

Потому что Smarty хранит скомпилированную PHP-версию вашего шаблона. Пока этот скомпилированный файл существует, Smarty не будет перечитывать `header.tpl`. После очистки Smarty заново скомпилирует шаблон из обновленного файла.

Кеш HTML здесь ни при чем, так как проблема в самом шаблоне, а не в данных.
</details>

---

### Вопрос 3 (продвинутый)
**В чем суть паттерна ViewModel и чем он отличается от простой передачи массива данных в шаблон?**

<details>
<summary>Ответ</summary>

**Суть ViewModel** — подготовить данные **специально для отображения**, чтобы логика форматирования не попадала в шаблон.

**Отличия от массива:**

| Аспект | Массив | ViewModel |
|--------|--------|-----------|
| Форматирование дат | В шаблоне или контроллере | Внутри ViewModel |
| Вычисления | Размазаны по коду | Инкапсулированы |
| Типизация | Нет (ассоциативный) | Есть (свойства класса) |
| IDE подсказки | Нет | Да |
| Переиспользование | Сложно | Легко |

**Пример:** вместо `{$user.birth_date|date_format:"%Y"}` в 10 шаблонах → один раз вычисляете `$user->age` в ViewModel.
</details>
