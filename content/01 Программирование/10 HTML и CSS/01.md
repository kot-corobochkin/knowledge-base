## Теория: Блочные и инлайн-элементы в HTML/CSS

Отличный вопрос! Эта проблема со стрелкой и заголовком — классический пример того, как важно понимать **типы отображения элементов** в CSS.

---

## 1. Что такое блочные и инлайн-элементы?

### Блочные элементы (block)

```css
display: block;
```

**Характеристики:**
- Занимают **100% ширины** родителя
- **Начинаются с новой строки**
- Имеют ширину и высоту (можно задать)
- Примеры: `<div>`, `<h1>`-`<h6>`, `<p>`, `<section>`, `<article>`

```html
<div>Я блочный</div>
<div>Я тоже блочный</div>
```

**Результат:**
```
Я блочный
Я тоже блочный        ← каждый на новой строке
```

### Инлайн-элементы (inline)

```css
display: inline;
```

**Характеристики:**
- Занимают **только нужную ширину**
- **Не переносят на новую строку**
- Нельзя задать ширину и высоту
- Примеры: `<span>`, `<a>`, `<strong>`, `<em>`

```html
<span>Я инлайн</span>
<span>Я тоже инлайн</span>
```

**Результат:**
```
Я инлайн Я тоже инлайн    ← в одной строке
```

### Инлайн-блок (inline-block)

```css
display: inline-block;
```

**Характеристики:**
- Комбинация лучших свойств
- Не переносит на новую строку (как inline)
- Можно задать ширину и высоту (как block)
- Идеально для кнопок, иконок с текстом

## 2. Ваша проблема в деталях

### Что у вас было:

```html
<summary>
    <h5>Список ошибок</h5>
</summary>
```

**Визуально:**
```
▶
Список ошибок
```

### Почему так произошло?

| Элемент | Тип отображения | Что делает |
|---------|----------------|------------|
| `<summary>` | по умолчанию `display: block` | Содержит стрелку |
| `<h5>` | по умолчанию `display: block` | Занимает всю ширину, переносит текст на новую строку |

**Схематично:**
```
┌─────────────────────────────────────────────┐
│ <summary>                                    │
│   ▶                                          │ ← стрелка
│   ┌───────────────────────────────────────┐ │
│   │ <h5>Список ошибок</h5>                │ │ ← h5 начал новую строку
│   └───────────────────────────────────────┘ │
│ </summary>                                   │
└─────────────────────────────────────────────┘
```

---

## 3. Решение: `display: inline-block`

```css
summary h5 {
    display: inline-block;
    margin: 0;
}
```

**Что изменилось:**

```
┌─────────────────────────────────────────────┐
│ <summary>                                    │
│   ▶ Список ошибок                            │ ← теперь в одной строке
│ </summary>                                   │
└─────────────────────────────────────────────┘
```

**Почему работает:**
- `inline-block` позволяет h5 **не переносить строку**
- Но при этом сохраняет возможность задавать отступы (margin, padding)

---

## 4. Полная аналогия для понимания

Представьте **железнодорожные вагоны**:

| Тип | Аналогия |
|-----|----------|
| **block** | Каждый вагон занимает весь путь → следующий вагон только после того, как предыдущий проехал всю дистанцию |
| **inline** | Вагоны цепляются друг за другом в одну линию |
| **inline-block** | Вагоны цепляются в линию, но каждый может быть разной длины |

Ваша проблема: `<h5>` был как поезд, который занимает ВЕСЬ путь, поэтому стрелка осталась на предыдущей станции.

---

## 5. Другие способы решения

### Способ 1: Убрать h5 полностью

```html
<style>
    summary {
        font-size: 1.25rem;
        font-weight: bold;
    }
</style>
<summary>Список ошибок</summary>
```

### Способ 2: Использовать span вместо h5

```html
<summary>
    <span style="font-size: 1.25rem; font-weight: bold;">Список ошибок</span>
</summary>
```

### Способ 3: Применить inline-block ко всему summary

```css
summary {
    display: inline-block;
}
```

## 6. Таблица сравнения display-значений

| Значение | Новая строка | Можно задать width/height | Примеры |
|----------|-------------|---------------------------|---------|
| `block` | ✅ Да | ✅ Да | div, h1, p |
| `inline` | ❌ Нет | ❌ Нет | span, a, strong |
| `inline-block` | ❌ Нет | ✅ Да | кнопки, иконки |
| `flex` | ✅ Да | ✅ Да | современные макеты |
| `none` | элемент скрыт | — | — |

## 3 Проверочных вопроса

### Вопрос 1 (базовый)
**Почему `<h5>` переносит текст на новую строку, а `<span>` — нет?**

<details><summary>Ответ</summary>

Потому что `<h5>` по умолчанию имеет `display: block`, а `<span>` — `display: inline`.

Блочный элемент всегда начинается с новой строки и занимает всю доступную ширину. Инлайн-элементы располагаются в строку друг за другом.
</details>


---

### Вопрос 2 (ситуационный)
**У вас есть строка: `▶ Список ошибок`. Нужно, чтобы при наведении мыши фон становился серым только у текста, а стрелка оставалась без фона. Как реализовать?**

<details>
<summary>Ответ</summary>

```html
<style>
    summary {
        list-style: none;  /* убираем стандартную стрелку */
        cursor: pointer;
    }
    summary::before {
        content: '▶';
        margin-right: 8px;
    }
    summary span {
        padding: 5px;
    }
    summary span:hover {
        background-color: #f0f0f0;
    }
</style>

<details>
    <summary><span>Список ошибок</span></summary>
    <div>Контент</div>
</details>
```

Идея: стрелка через псевдоэлемент `::before`, текст в `<span>` с отдельным hover-стилем.
</details>

---

### Вопрос 3 (продвинутый)
**В чем разница между `inline-block` и `inline-flex`? Когда что использовать?**

<details>
<summary>Ответ</summary>

| Свойство | Поведение в строке | Внутреннее поведение |
|----------|-------------------|---------------------|
| `inline-block` | Не переносит строку | Обычное блочное (блоки идут вертикально) |
| `inline-flex` | Не переносит строку | Flex-контейнер (элементы выравниваются flex-свойствами) |

**Когда что использовать:**
- `inline-block` — для простых элементов (кнопка с иконкой, тег)
- `inline-flex` — когда нужны flex-свойства внутри (выравнивание по центру, порядок элементов)

**Пример для `inline-flex`:**
```css
.menu-item {
    display: inline-flex;
    align-items: center;
    gap: 8px;
}
```
</details>

## Итог

**Ваша проблема:** `<h5>` — блочный элемент, он переносит строку.

**Решение:**
```css
summary h5 {
    display: inline-block;  /* ← волшебная строка */
    margin: 0;
}
```

**Запомните:** 
- `block` → новая строка
- `inline` → в строке, но нельзя задать размеры
- `inline-block` → лучшее из двух миров
