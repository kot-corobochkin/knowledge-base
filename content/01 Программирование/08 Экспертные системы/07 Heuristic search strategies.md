## 1. Введение в эвристики

**Эвристики** — это приближенные методы или «эмпирические правила», используемые для минимизации процесса поиска. Они применяются к двум категориям задач:

| Категория | Описание | Примеры |
| --- | --- | --- |
| Нет точного алгоритма | Необходимо найти приближенное, удовлетворительное решение | Компьютерное зрение, распознавание речи |
| Точное решение известно, но невыполнимо | Вычислительная сложность делает точное решение непрактичным | Шахматы |

**Эвристическая функция:** Отображает состояния задачи в числа, которые приблизительно направляют поиск к цели.

**Примеры эвристических функций:**
- **Задача 8:** Расстояние Хэмминга (количество фишек, находящихся не на своих местах).
- **Шахматы:** Материальное преимущество (разница в ценности фигур).

**Цель эвристик:**
- Улучшить качество исследуемых путей.
- Уменьшить пространство поиска.
- Эффективно находить решения.

**Аналогия:** Искать розовый двухэтажный дом в городе гораздо эффективнее, чем проверять каждый адрес (слепой поиск).

## 2. Типы эвристических методов поиска

## 2.1 «Сгенерируй и проверь» (Generate and Test)

Простейший эвристический подход.

**Алгоритм:**
- Сгенерировать возможное решение.
- Проверить, является ли оно решением.
- Если решение найдено — завершить работу, иначе вернуться к шагу 1.

**Вариации:**
- **Случайная генерация:** Решение может никогда не быть найдено.
- **Систематическая генерация:** Гарантирует нахождение решения, если оно существует, но может быть очень медленной для больших пространств.

**Наилучшее применение:** Задачи с ограниченным пространством поиска, где возможен исчерпывающий поиск.

## 2.2 Поиск «Лучший-первый» (Best First Search)

Вариант поиска в ширину, использующий эвристическую (оценочную) функцию для определения, какой узел развернуть следующим.

**Ключевые понятия:**
- **Оценочная функция:** Указывает, насколько далеко узел находится от целевого узла (целевой узел имеет значение 0).
- **ИЛИ-граф:** Каждая ветвь представляет собой альтернативный путь решения задачи.
- **Список OPEN:** Узлы, которые были сгенерированы, но еще не исследованы.
- **Список CLOSED:** Узлы, которые уже были исследованы (чтобы избежать повторения путей).

**Алгоритм:**
- Поместить начальный узел в список START.
- Если START пуст или START является целью, завершить работу.
- Удалить первый узел из START, назвать его a.
- Если a — цель, завершить работу с успехом.
- Иначе сгенерировать всех преемников, отсортировать их по оставшемуся расстоянию до цели и добавить в начало START.
- Перейти к шагу 2.

**Характеристики:**
- Объединяет преимущества поиска в ширину (не застревает в тупиках) и поиска в глубину (находит решение без развертывания всех ветвей).
- На каждом шаге выбирает самый перспективный узел из всех сгенерированных преемников.
- Реализуется с помощью очереди с приоритетом.

## 2.3 Восхождение на холм (Hill Climbing)

Алгоритм дискретной оптимизации, который непрерывно движется в направлении увеличения значения целевой функции.

**Ключевые принципы:**
- Использует единственное текущее состояние (не дерево поиска).
- Смотрит только на непосредственных соседей.
- Завершается при достижении пика (ни один сосед не имеет большего значения).
- Путь решения не важен; важна только конечное состояние.

**Области применения:**
- Проектирование схем
- Составление расписаний
- Маршрутизация транспорта
- Автоматическое программирование

**Алгоритм:**
- Поместить начальный узел в список START.
- Если START пуст или START является целью, завершить работу.
- Удалить первый узел из START, назвать его a.
- Если a — цель, завершить работу с успехом.
- Иначе сгенерировать всех преемников, отсортировать их по оставшемуся расстоянию до цели и добавить в начало START.
- Перейти к шагу 2.

**Ограничения:**

| Проблема | Описание |
| --- | --- |
| Локальный максимум | Пик, который выше всех соседей, но ниже глобального максимума; алгоритм останавливается преждевременно. |
| Плато | Плоская область, где все соседи имеют одинаковое значение; трудно выбрать направление. |
| Хребет | Узкая возвышенность, требующая осторожного продвижения; движение в любом направлении может привести к спуску. |

**Способы преодоления ограничений:**
- Вернуться к более раннему узлу и попробовать другое направление (для локальных максимумов).
- Сделать большой «прыжок» для исследования новой области пространства (для плато).
- Применить несколько правил перед проверкой (для хребтов).

## 2.4 Имитация отжига (Simulated Annealing)

Сочетает восхождение на холм со случайным поиском для преодоления проблемы локальных максимумов.

**Ключевые принципы:**
- Принимаются ходы, которые улучшают ситуацию.
- Худшие ходы принимаются с вероятностью **P = e^(–ΔE/KT)**

ΔE = положительное изменение «энергетического уровня»
- T = температура
- K = постоянная Больцмана

- Вероятность уменьшается с «плохостью» хода.
- График охлаждения (annealing schedule): постепенное снижение температуры от высокого значения.

**Отличия от восхождения на холм:**
- Поддерживается график охлаждения.
- Принимаются ходы к худшим состояниям.
- Запоминается лучшее достигнутое состояние (а не только текущее).

**Применение:**
- Размещение микросхем (VLSI layout)
- Составление заводских расписаний
- Крупномасштабные оптимизационные задачи

**Алгоритм:**
- Вычислить значение начального состояния. Сделать его текущим и лучшим.
- Инициализировать T согласно графику охлаждения.
- Повторять, пока решение не найдено или не закончились операторы:
a. Применить оператор для получения нового состояния.
b. Вычислить ΔE = value(текущее) – value(новое). Если новое состояние — цель или лучше текущего, сделать его текущим и запомнить как лучшее.
c. Если новое состояние хуже, сделать его текущим с вероятностью P.
d. Изменить T согласно графику охлаждения.
- Вернуть лучшее состояние.

## 2.5 Алгоритм A*

Вариация поиска «Лучший-первый», которая объединяет стоимость достижения узла и оценочную стоимость пути до цели.

**Оценочная функция:**
- **f(n) = g(n) + h(n)**

g(n) = стоимость пути от начального узла до узла n
- h(n) = оценочная стоимость наименьшего пути от n до цели
- f(n) = оценочная стоимость наименьшего решения, проходящего через n

**Ключевые понятия:**
- **Функция стоимости:** Ресурсы, затраченные на достижение узла.
- **Оценочная функция:** Оценка будущей стоимости.
- **Число пригодности (fitness number):** Сумма оценочной функции и функции стоимости.

**Алгоритм:**
- Поместить начальный узел в список START.
- Если START пуст или START является целью, завершить работу.
- Удалить первый узел из START, назвать его a.
- Если a — цель, завершить работу с успехом.
- Иначе сгенерировать всех преемников, вычислить для каждого число пригодности (g + h), отсортировать по числу пригодности и добавить в START.
- Перейти к шагу 2.

**Свойства:**
- **Полнота:** Да (всегда завершается с решением, если оно существует).
- **Оптимальность:** Да (если h(n) является допустимой).
- Поддерживает списки OPEN и CLOSED.

**Допустимая эвристика:** Никогда не переоценивает стоимость достижения цели (оптимистична). Пример: расстояние по прямой линии в задаче поиска маршрута.

## 2.6 Графы И-ИЛИ (AND-OR Graphs, алгоритм AO*)

Используется для задач, которые могут быть декомпозированы на более мелкие подзадачи.

**Ключевые понятия:**
- **ИЛИ-граф:** Решение любой одной подзадачи решает всю задачу.
- **И-дуга:** Все узлы-преемники должны быть решены, чтобы решить задачу.

**Пример:** Генерация деревьев разбора для английских предложений
- S → NP VP (И: нужны и именная, и глагольная группы)
- NP → N (ИЛИ: либо существительное)
- NP → art N (ИЛИ: артикль + существительное)

**Принципы алгоритма AO*:**
- Определяет все поддеревья, которые должны быть решены.
- Помечает узлы как «решенные» или «нерешенные».
- Решение найдено, когда начальный узел помечен как решенный.

| Критерий | BFS | UCS | DFS | DLS | IDS | BS |

| Время | b^d | b^d | b^m | b^l | b^d | b^(d/2) |
| Память | b^d | b^d | b^m | b^l | b^d | b^(d/2) |
| Оптимальность? | Да | Да | Нет | Нет | Да | Да |
| Полнота? | Да | Да | Нет | Да (если l ≥ d) | Да | Да |

*Где:*
- *b = коэффициент ветвления*
- *d = глубина ближайшего целевого узла*
- *m = максимальная глубина дерева поиска*
- *l = ограничение глубины*
- *BS = двунаправленный поиск*

## 6. Заключение

Эвристические стратегии поиска обеспечивают значительное улучшение по сравнению с методами вслепую за счет использования знаний, специфичных для предметной области, для направления процесса исследования. Рассмотренные методы варьируются от простых подходов, таких как «Сгенерируй и проверь», до сложных алгоритмов, таких как A* и имитация отжига. Выбор эвристического поиска зависит от характеристик задачи и включает компромиссы между оптимальностью, полнотой и вычислительными ресурсами. Поиск с противником расширяет эти концепции на конкурентные среды, такие как игровые задачи, где алгоритм MINIMAX обеспечивает оптимальное принятие решений против оппонента.
