[[09 Knowledge Representation]]

## 7. Types of Knowledge Representational Schemes

## 7.1 Formal Logic

Formal logic is a language with concrete rules, no ambiguity, allowing unambiguous communication and processing.

**Key Concepts:**
- **Syntax:** Rules for constructing legal sentences (symbols, combination rules)
- **Semantics:** Meaning of sentences; how to interpret them

**Formal Logic = Formal Language + Semantics**
7.1.1 Propositional Logic
The simplest logic, using propositions that are either true or false.

**Components:**
- **Propositions:** Statements that are true or false (e.g., "Ram is a boy")
- **Simple proposition:** Cannot be broken down (atomic)
- **Compound proposition:** Contains other propositions as parts

**Logical Connectives:**

| Connective | Symbol | Meaning |
| --- | --- | --- |
| Conjunction | ∧ | AND (both must be true) |
| Disjunction | ∨ | OR (at least one true) |
| Implication | ⇒ | IF...THEN (antecedent → consequent) |
| Equivalence | ⇔ | Biconditional (if and only if) |
| Negation | ¬ | NOT (opposite) |

**Truth Table (P and Q):**

| P | Q | ¬P | P∧Q | P∨Q | P⇒Q | P⇔Q |
| --- | --- | --- | --- | --- | --- | --- |
| F | F | T | F | F | T | T |
| F | T | T | F | T | T | F |
| T | F | F | F | T | F | F |
| T | T | F | T | T | T | T |

**Properties/Rules:**

| Rule | Formula |
| --- | --- |
| Commutative | E∧F ⇔ F∧E, E∨F ⇔ F∨E |
| Distributive | E∧(F∨G) ⇔ (E∨F)∨(E∧G), E∨(F∧G) ⇔ (E∨F)∧(E∨G) |
| Associative | E∧(F∧G) ⇔ (E∧F)∧G, E∨(F∨G) ⇔ (E∨F)∨G |
| De Morgan's | ¬(E∧F) ⇔ ¬E∨¬F, ¬(E∨F) ⇔ ¬E∧¬F |
| Negation | ¬(¬E) ⇔ E |

**Strengths:**
- Easy to represent real-world knowledge
- Simple, declarative
- Permits conjunctive/disjunctive/partial/negative information

**Weaknesses:**
- Very limited expressive power
- Cannot represent properties of objects
- Cannot represent relationships between similar facts
- Cannot represent categorical syllogisms

7.1.2 Predicate Logic
More expressive than propositional logic; deals with objects, attributes, and relationships.

**Components:**

| Component | Description | Example |
| --- | --- | --- |
| Constants | Specific objects | Ramesh, 2, March |
| Functions | Map arguments to values | brotherof, father(John) |
| Variables | Placeholders for objects | x, y, z |
| Predicates | Properties or relations (true/false) | man(Ramesh), Brother(Mukesh, Suresh) |
| Quantifiers | Specify variable scope | ∀ (universal), ∃ (existential) |

**Quantifiers:**

| Quantifier | Symbol | Meaning | Example |
| --- | --- | --- | --- |
| Universal | ∀ | For all | ∀x (MAN(x) → MORTAL(x)) |
| Existential | ∃ | There exists | ∃x Crown(x) ∧ OnHead(x, John) |

**Atomic Sentence:** Predicate symbol with terms in parentheses
- Example: Brother(Mukesh, Suresh)
- Complex: Married(Father(Richard), Mother(John))

**Types of Predicate Logic:**

| Type | Description |
| --- | --- |
| Zero Order | No arguments |
| First Order (FOPL/FOPC) | One argument; more expressive than propositional logic |
| Higher Order | Arguments can be predicates or functions |

**Examples:**
- "Ram likes apples": likes(Ram, apples)
- "Today is wet": is_wet(today)
- "Every man is mortal": ∀x (MAN(x) → MORTAL(x))
- "Every Monday and Wednesday, I go to Ram's house for dinner":
∀X ((day_of_week(X, Monday) ∧ day_of_week(X, Wednesday)) → (go_to(me, house_of(john)) ∨ eat(me, dinner)))
